Architecture

Core layers

  • app.py holds the legacy bootstrap and top-level runtime helpers.
  • views.py is the public web facade for HTTP and WebSocket surfaces.
  • utils.py contains shared helpers used by app, server, and agent code.
  • dns.py centralizes resolver transport, parsing, and lookup logic.

Runtime nodes

  • master.py starts the master node.
  • agent.py starts the agent node and enrollment loop.
  • server.py exposes the scan API and persistence layer.
  • frontend/ is the Vue 3 user interface bundled on top of the API.
The web layer is intentionally separated from utility helpers so the application can stay readable and the documentation can point to stable responsibilities instead of one large monolith.

Installation

PyPI

python3 -m pip install porthound

Local checkout

python3 -m venv env
env/bin/python -m pip install --upgrade pip
env/bin/python manage.py
PortHound requires Python 3.12 or newer and the wsbuilder runtime.

Runtime modes

Master

porthound --role master --host 0.0.0.0 --port 45678 --db-path ./Master.db

Used to coordinate scans, API access, and agent orchestration.

Agent

porthound --role agent --master http://127.0.0.1:45678 --agent-id <id> --agent-token <token>

Pulls tasks from a master and executes work on the local runtime.

Standalone

Standalone mode keeps the local master-like workflow but does not require a remote agent enrollment chain.

Configuration

General

  • PORTHOUND_HOST and PORTHOUND_PORT control bind address and port.
  • PORTHOUND_DB_PATH selects the active SQLite database.
  • PORTHOUND_ROLE sets the runtime role: master, agent, or standalone.
  • PORTHOUND_API_TOKEN protects admin endpoints.

TLS and agent

  • PORTHOUND_TLS_ENABLED enables TLS for the master node.
  • PORTHOUND_CA or PORTHOUND_CA_ONELINE define the CA chain.
  • PORTHOUND_AGENT_CERT and PORTHOUND_AGENT_KEY enable client certificates.
  • PORTHOUND_AGENT_TOKEN is the enrollment secret for agent mode.

CLI

Useful flags

  • --role, --host, --port, --db-path
  • --master, --agent-id, --agent-token
  • --dns-resolvers, --dns-timeout-seconds, --dns-use-system-resolver

Environment override pattern

export PORTHOUND_ROLE="master"
export PORTHOUND_HOST="0.0.0.0"
export PORTHOUND_PORT="45678"
export PORTHOUND_DNS_RESOLVERS="udp://10.0.0.2:53,tcp://10.0.0.2:53"

DNS and records

Resolver transports

  • udp:// for plain DNS
  • tcp:// for TCP DNS
  • dot:// or tls:// for DNS over TLS
  • doh:// for DNS over HTTPS

PortHound DNS config

export PORTHOUND_DNS_RESOLVERS="udp://10.0.0.2:53,tcp://10.0.0.2:53,dot://dns.example.org:853,doh://dns.example.org/dns-query"
export PORTHOUND_DNS_TIMEOUT_SECONDS="1.4"
export PORTHOUND_DNS_USE_SYSTEM_RESOLVER="1"

Record types

  • A: IPv4 address.
  • AAAA: IPv6 address.
  • CNAME: alias to another name.
  • MX: mail exchange host.
  • NS: authoritative nameserver.
  • TXT: arbitrary text and verification metadata.
  • SRV: service discovery records.
  • PTR: reverse DNS mapping.
  • CAA: certificate issuance policy.

Local DNS example

from dns import build_local_dns_server

records = {
    "scan.local": ["10.10.0.10"],
    "dashboard.local": ["10.10.0.11", "::1"],
}

dns_server = build_local_dns_server(records=records, host="0.0.0.0", port=5300)
dns_server.start()
Port 53 usually needs administrator privileges. In labs, using 5300 is simpler and you can route clients to it through DHCP or a port forward.

Client configuration

Linux

sudo resolvectl dns eth0 10.10.0.2
sudo resolvectl domain eth0 '~local'

Windows

  • Configure DNS manually on the adapter.
  • Use 10.10.0.2 as preferred DNS and 10.10.0.3 as alternate DNS.
  • Push DHCP option 6 and search domain option 15 when using a router or DHCP server.

macOS

networksetup -setdnsservers "Wi-Fi" 10.10.0.2 10.10.0.3

Router / DHCP

  • Set the DNS handed out by DHCP to your internal resolver.
  • Add the private suffix to the search domain list.
  • Prefer the internal resolver over the ISP DNS for private zones.

API and WebSocket

Local endpoints

  • http://HOST:PORT/ for the UI and API base.
  • http://HOST:PORT/ws/ for the WebSocket endpoint.
  • GET /protocols/ to inspect active runtime protocols.

Behavior

  • Text frames are echoed back.
  • Binary frames are echoed with a binary prefix.
  • Aliased messages can be persisted in the local demo/chat storage.

Troubleshooting

  • If a DNS resolver is unreachable, move to the next entry in PORTHOUND_DNS_RESOLVERS.
  • If you cannot bind to port 53, use 5300 in the local DNS example.
  • If admin endpoints reject requests, set PORTHOUND_API_TOKEN or use loopback only.
  • If the agent cannot connect to the master, verify that PORTHOUND_MASTER uses http:// and a reachable host.
  • If TLS validation fails, check the CA file or the inline CA value before changing client settings.

Security

  • Use PortHound only on assets you own or have explicit authorization to test.
  • Protect the master API with PORTHOUND_API_TOKEN in non-loopback deployments.
  • Prefer internal DNS for private zones and avoid exposing internal names to public resolvers.
  • Keep agent enrollment tokens secret and rotate them when an agent is reissued.

Code layout

  • utils.py: shared helpers for time, roles, and target normalization.
  • views.py: public web facade for the HTTP/WebSocket surface.
  • dns.py: resolver transports and DNS record lookup helpers.
  • app.py: application bootstrap and legacy compatibility layer.
  • server.py: scan API and target persistence logic.
