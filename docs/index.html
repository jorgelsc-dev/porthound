PortHound

Escaneo local con UI, API y SQLite.

PortHound empaqueta escaneo activo, banner grabbing, mapa, analitica y persistencia local en un solo runtime Python. El flujo soportado del proyecto es standalone, servido por el mismo proceso sobre wsbuilder.

Host efectivo 127.0.0.1
Puerto efectivo 45678

Mapa del runtime

1. Targets
Redes, protocolos y rangos se registran en SQLite.
2. Scanners
TCP, UDP, ICMP y SCTP procesan el trabajo activo.
3. Persistencia
Puertos, banners, tags y favicons quedan versionados localmente.
4. UI/API
Dashboard, mapas, analitica y WebSocket leen el mismo estado.

Bloques principales

Escaneo
Targets con acciones individuales o bulk sobre protocolos soportados por el runtime.
Catalogos
Reglas regex, probes y presets de IP se editan por API y por ficheros seed.
Observabilidad
Dashboard, chart analytics, mapa y feed de ataques comparten el mismo backend.
GeoIP
Seeds versionados en data/ se importan a la base activa al arrancar.

Inicio rapido

Instalar 
python -m pip install porthound4
Arrancar 
porthound --db-path Standalone.db
Dashboard 
http://127.0.0.1:45678
Estado rapido 
curl http://127.0.0.1:45678/api/dashboard/

El launcher standalone fija 127.0.0.1:45678. Los flags --host y --port no cambian ese bind en el flujo publico del paquete.

Superficie API y WS

  • /targets/, /target/, /target/action/ para control de trabajo.
  • /ports/*, /banners/, /tags/* y /favicons/ para resultados.
  • /api/dashboard/, /api/charts/analytics y /api/map/scan para snapshots de UI.
  • /api/catalog/* para reglas regex, probes e IP presets.
  • /api/ip/domains/, /api/ip/ttl-path/ y /api/ip/intel/ para enrichment.
  • /api/ws/* y WS /ws/ para clientes conectados, ping, broadcast y snapshots.

Seguridad y alcance

  • Sin PORTHOUND_API_TOKEN, las acciones administrativas solo aceptan clientes loopback.
  • PORTHOUND_API_TOKEN se envia por Authorization: Bearer o X-API-Key.
  • PORTHOUND_API_REQUIRE_TOKEN=1 debe usarse junto con PORTHOUND_API_TOKEN.
  • La distribucion CA y el enrollment por certificado estan deshabilitados en el runtime actual.
  • Usa PortHound solo para auditorias autorizadas.
